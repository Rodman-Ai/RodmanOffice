# LeoCRM

An AI-first lead-generation CRM that lives in your Google Workspace.
Sign in with Google, and LeoCRM provisions a `LeoCRM Workspace` Google Sheet
and a `LeoCRM` Drive folder in your account — that's the entire database.
Outbound emails are generated by Claude, edited by you, and sent through your
Gmail. Runs on desktop and mobile (responsive web app, installable as a PWA).

## Features

- **Google Workspace login** — OAuth with Sheets, Drive, and Gmail-send scopes.
- **Sheets-as-database** — `Contacts`, `Leads`, `Campaigns`, `Emails`,
  `Templates`, `Config` tabs are auto-created on first sign-in.
- **AI email composer** — Claude generates a personalized cold email from a
  goal + tone + the prospect's profile. You edit, then send.
- **Gmail send + history** — Emails are sent via the user's Gmail; every send
  is logged back into the `Emails` tab and attached to the contact.
- **Pipeline** — drag-free kanban that nudges leads through `new → contacted →
  engaged → qualified → won/lost`. Sending an email auto-bumps `new` to
  `contacted` and updates `lastContactedAt`.
- **Campaigns + templates** — group sends by goal/audience, reusable email
  skeletons with optional AI-prompt seeds.
- **Drive uploads** — attachment helper writes into the LeoCRM folder.
- **Responsive UI** — sidebar on desktop, bottom-nav on mobile, PWA manifest
  so it installs on iOS/Android home screens.

## Architecture

```
Next.js 14 (App Router, TS) ────────────────────────┐
  ├── /api/* route handlers ── googleapis (Sheets, Drive, Gmail)
  ├── /api/ai/generate ─────── @anthropic-ai/sdk
  └── NextAuth (Google provider, JWT) with token refresh
```

No external database — all CRM rows live in the user's spreadsheet, all
documents in the user's Drive folder. The app is stateless beyond the
NextAuth JWT cookie.

## Setup

### 1. Google Cloud project

1. Create / pick a project at <https://console.cloud.google.com/>.
2. Enable APIs: **Google Sheets API**, **Google Drive API**, **Gmail API**.
3. OAuth consent screen → "Internal" (Workspace) or "External" + add yourself
   as a test user.
4. Credentials → Create OAuth client → "Web application".
   - Authorized JavaScript origins: `http://localhost:3000`
   - Authorized redirect URIs: `http://localhost:3000/api/auth/callback/google`

### 2. Anthropic API key

Grab a key from <https://console.anthropic.com>. The default model is
`claude-sonnet-4-6` — override with `ANTHROPIC_MODEL` if needed. (If no key
is set, Compose falls back to a basic template so you can still try the flow.)

### 3. Environment

```bash
cp .env.example .env.local
# fill in NEXTAUTH_SECRET (openssl rand -base64 32),
# GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET, ANTHROPIC_API_KEY
```

Optional: set `GOOGLE_WORKSPACE_DOMAIN=acme.com` to restrict sign-in to a
single Workspace domain.

### 4. Run

```bash
npm install
npm run dev
# open http://localhost:3000
```

First sign-in will:

1. Create a spreadsheet titled `LeoCRM Workspace` in your Drive.
2. Create a folder `LeoCRM` and move the spreadsheet inside it.
3. Add the six tabs and headers.

You can verify in `/settings`, which links straight to both.

### 5. Deploy

Two paths, both via GitHub Actions:

**A) One-click via the bundled Vercel workflow** (`.github/workflows/deploy.yml`).
Configure once, then every push to `main` ships to production and every other
branch ships to a preview URL.

1. `npm i -g vercel && vercel link` in the repo (or import the repo at
   <https://vercel.com/new>). This creates `.vercel/project.json` with `orgId`
   and `projectId`.
2. Add the following **Repository Secrets** (Settings → Secrets and variables
   → Actions → New repository secret):
   - `VERCEL_TOKEN` — a personal token from <https://vercel.com/account/tokens>
   - `VERCEL_ORG_ID` — from `.vercel/project.json`
   - `VERCEL_PROJECT_ID` — from `.vercel/project.json`
3. In the Vercel project's **Environment Variables**, add the same values
   you set in `.env.local` (`NEXTAUTH_SECRET`, `GOOGLE_CLIENT_ID`,
   `GOOGLE_CLIENT_SECRET`, `ANTHROPIC_API_KEY`, `NEXTAUTH_URL` set to the
   production URL).
4. In Google Cloud Console, add your Vercel production + preview URLs to
   **Authorized JavaScript origins** and `…/api/auth/callback/google` to
   **Authorized redirect URIs**.
5. Push. The `Deploy to Vercel` workflow runs on every push and posts the
   deploy URL to the job summary.

If the secrets aren't set, the deploy workflow skips itself and prints
instructions in the job summary — CI still runs (typecheck + build).

**B) Direct Vercel/Netlify** — drop the GitHub Action and just import the repo
in the Vercel/Netlify UI; both detect Next.js and "just work" with the env
vars from step 3.

**C) Cloudflare Pages** — supports Next.js with Functions out of the box.
Connect the repo in the Cloudflare dashboard, set the same env vars, and
push.

### Why not GitHub Pages?

GitHub Pages serves static files only. LeoCRM needs a server to (a) handle
NextAuth's OAuth callback and refresh tokens, (b) call Google Sheets/Drive/
Gmail with the user's access token, and (c) call Claude with the
server-side `ANTHROPIC_API_KEY` (which must never ship to the browser). The
bundled `.github/workflows/pages.yml` publishes a marketing/landing page to
GitHub Pages that links to your real, server-hosted app — useful as a free
public URL that says what LeoCRM is.

## Mobile

The app is a responsive PWA — visit on iOS Safari or Android Chrome and use
"Add to Home Screen" to install. The bottom nav handles primary navigation;
the sidebar appears at `md:` breakpoints and up.

## Data model (Google Sheet tabs)

| Tab        | Columns                                                                                                   |
| ---------- | --------------------------------------------------------------------------------------------------------- |
| Contacts   | id, name, email, company, role, phone, linkedin, tags, notes, createdAt, updatedAt                        |
| Leads      | id, contactId, source, stage, score, value, owner, lastContactedAt, nextActionAt, nextAction, notes, …    |
| Campaigns  | id, name, goal, audience, status, templateId, sentCount, repliedCount, createdAt                          |
| Emails     | id, contactId, campaignId, subject, body, sentAt, status, aiGenerated, prompt, threadId                   |
| Templates  | id, name, subject, body, aiPrompt, createdAt                                                              |
| Config     | key, value                                                                                                |

You can edit rows directly in Sheets; the app will pick up changes on the
next read.

## Security notes

- The OAuth scope is `drive.file`, not full Drive — LeoCRM only sees files it
  created.
- Gmail uses `gmail.send` + `gmail.compose`, no inbox read.
- Tokens never leave the server (NextAuth JWT cookie is HTTP-only).
- Anthropic calls happen server-side; your API key is never shipped to the
  browser.
